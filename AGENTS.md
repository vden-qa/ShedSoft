# AGENTS — Руководство для контрибьюторов

Краткое и практичное руководство по вкладу в проект ShedSoft.

## Структура проекта
- src/ — основной исходный код Python-проекта
- tests/ — модульные и интеграционные тесты
- docs/ — документация и заметки по архитектуре
- config/ — конфигурационные файлы и шаблоны
- scripts/ — вспомогательные скрипты для запуска/развертывания

(Пути относительны корню проекта: /home/vden/PycharmProjects/ShedSoft)

## Команды разработки
- python -m venv .venv — создать виртуальное окружение
- source .venv/bin/activate — активировать среду (Linux)
- pip install -r requirements.txt — установить зависимости
- pytest -q — запустить все тесты
- pytest tests/path/to/test_file.py::test_name -q — запустить конкретный тест
- flake8 . — проверить стиль кода

## Стиль кода и тесты
- Используется PEP8, 4 пробела отступа, max-line-length 88 (Black compatible)
- Именование: snake_case для функций и переменных, PascalCase для классов
- Тесты: pytest; файлы тестов называются test_*.py; тестовые функции test_*()
- Цель покрытия: >= 80% для бизнес-логики; покрывать крайние случаи и обработку ошибок

## VCS: коммиты и PR
- Коммиты в настоящем репозитории короткие и осмысленные: "feat: add X", "fix: handle Y edge case", "chore: update deps"
- Перед PR: убедиться, что все тесты проходят и flake8 чист
- PR template: описать суть изменений, шаги для проверки, связанные issue, скриншоты при необходимости
- Мерж через squash-merge; в описание PR краткий changelog

## Безопасность и конфигурация
- Не коммитить секреты; для локальной разработки использовать .env и .env.example
- Конфигурация в config/; перечислить ключи в docs/config.md

## Как помочь — чеклист
1. Форкнуть и создать ветку feature/your-short-desc
2. Реализовать изменения + тесты
3. Запустить pytest и flake8
4. Открыть PR с описанием и ссылкой на issue
